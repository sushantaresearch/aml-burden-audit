README_data_paths.txt: where the analysis scripts expect data
===============================================================

All CELL_*.py files in this folder are Google Colab notebook cells
exported as plain Python. They mount Google Drive at /content/drive
and use the following locations (paths taken verbatim from the
scripts themselves):

1. /content/drive/MyDrive/Kaggle/gnn_outputs/
   Output directory for every derived artifact: detector score CSVs
   (*_val_scores*.csv, *_test_scores*.csv), reweighting ladders
   (amlworld_gnn_4c_*_ladder.csv), audit and bootstrap CSVs
   (bh_family_results.csv, cluster_bootstrap_*.csv,
   entity_burden_hismall.csv, customs_*.csv), figures
   (fig_bh_forest.*, fig_dose_response*.*), and model checkpoints
   (checkpoint_4c_*.tar). The customs scripts (CELL_CX1 to CELL_CX3)
   write here when Drive is mounted and silently fall back to the
   local working directory when it is not.

2. /content/drive/MyDrive/Kaggle/aml_cache/
   Cached preprocessed AMLworld HI-Small inputs:
   - Small_HI_formatted_transactions.csv (5,078,345 rows; produced
     by the Multi-GNN data formatting step from the IBM AMLworld
     HI-Small raw transactions CSV)
   - g_sorted_hismall.npy (per-transaction entity-group codes,
     aligned to the formatted CSV row order; 0 = S, 1 = M, 2 = L)
   - w_edge_4c_w{W}.npy (per-train-edge loss weights for the
     Front 4C reweighting dose, W in 1.5/2.5/4.0; length equals the
     3,248,921 training edges)

3. /content/Multi-GNN
   Clone of the IBM Multi-GNN reference implementation. The graph
   scripts import train_util.py, data_loading.py, models.py, and
   training.py from this tree and patch data_config.json at run
   time to point at the staged data.

4. /content/aml_data/<dataset>/formatted_transactions.csv
   Local (non-Drive) staging of the formatted AMLworld CSV used for
   graph construction; the scripts locate it by glob and refuse to
   run if zero or more than one candidate is found.

5. /content/kaggle.json
   Kaggle API credentials. CELL_CX4_auth_download.py uses them to
   fetch the IEEE-CIS fraud-detection dataset
   (train_transaction.csv) for the design-effect comparison cells.

6. https://github.com/Seondong/Customs-Declaration-Datasets.git
   (branch "en") Public synthetic customs import-declarations
   benchmark of Jeong et al. The customs scripts clone this
   repository and read df_syn_train_eng.csv, df_syn_valid_eng.csv,
   and df_syn_test_eng.csv from the cloned tree.

7. Bibliography utilities (CELL_BIB3 to CELL_BIB6) locate the .bib
   files anywhere under /content/drive/MyDrive/ by recursive glob,
   in the chain references_v2_autogenerated.bib ->
   references_v3_merged.bib -> references_v3_clean.bib.

Temporal split constants used throughout the AMLworld HI-Small
scripts: train edges 0 to 3,248,920; validation 3,248,921 to
4,214,444; test 4,214,445 to 5,078,344 (global transaction indices
on the time-sorted formatted CSV).

Nothing in results/ requires re-running these scripts: the CSVs in
results/ are the canonical outputs used in the paper. Re-running is
needed only to regenerate large release assets or to extend the
experiments.
